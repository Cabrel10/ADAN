DATA ARCHITECTURE
================================================================================

Real Data Flow:
  ADAN System
    ↓
  Paper Trading Monitor (every 5 minutes)
    ↓
  State File: /mnt/new_data/t10_training/phase2_results/paper_trading_state.json
    ↓
  RealDataCollector (reads fresh, no caching)
    ↓
  Dashboard (displays real-time data)

Key Components:
  - Paper Trading Monitor: Collects real trading data
  - State File: Shared JSON data store
  - RealDataCollector: Reads state file (always fresh)
  - Dashboard: Displays data with configurable refresh

================================================================================
TROUBLESHOOTING
================================================================================

Dashboard Shows Empty Data:
  1. Check if monitor is running: ps aux | grep paper_trading_monitor
  2. Start monitor if needed: python scripts/paper_trading_monitor.py

Dashboard Shows Old Data:
  1. Check state file: stat /mnt/new_data/t10_training/phase2_results/paper_trading_state.json
  2. Restart monitor: pkill -f paper_trading_monitor && python scripts/paper_trading_monitor.py

Connection Error:
  1. Create directory: mkdir -p /mnt/new_data/t10_training/phase2_results
  2. Set permissions: chmod 755 /mnt/new_data/t10_training/phase2_results
  3. Restart dashboard: python scripts/adan_btc_dashboard.py
