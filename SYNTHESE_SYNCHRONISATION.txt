🎓 CLARIFICATIONS TERMINOLOGIQUES
═════════════════════════════════════════════════════════════════════════════════

Source unique de vérité
  = Une seule place où les données existent
  = Pas de doublons ou de conflits
  = Exemple: 1 base de données pour les métriques (pas 3)

Synchronisation
  = Tous les systèmes ont les mêmes données
  = Pas de décalage ou d'incohérence
  = Exemple: Logger, metrics, trades tous à jour

Validation croisée
  = Vérifier avec 2-3 méthodes différentes
  = Comparer les résultats
  = Exemple: Calculer Sharpe 3 fois, comparer

Reprise sur erreur (retry)
  = Réessayer automatiquement si ça échoue
  = Jusqu'à N tentatives
  = Exemple: Trade échoue → Réessayer 3 fois

Centralisation
  = Tout au même endroit
  = Facile à trouver et à maintenir
  = Exemple: Tous les logs dans 1 fichier

Incohérence
  = Données différentes au même endroit
  = Problème à résoudre
  = Exemple: Sharpe = 1.5 ici, 2.3 là

Pipeline
  = Chaîne de traitement étape par étape
  = Chaque étape valide la précédente
  = Exemple: Action → Validation → Exécution → Log

Persistance
  = Garder les données après redémarrage
  = Sauvegarder dans une base de données
  = Exemple: Trades sauvegardés dans SQLite
